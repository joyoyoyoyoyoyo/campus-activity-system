# 工作分解结构（WBS）

> 围绕项目交付物展开的工作分解，是后续进度计划与任务分派的依据。

## 1. WBS 编号规则

`X.Y.Z` 形式，X 为一级交付物，Y 为子交付物，Z 为具体工作包（Work Package）。

## 2. 总览

```
1. 项目管理
2. 需求与分析
3. 设计
4. 实现
5. 测试与验证
6. 配置管理
7. 交付与总结
```
