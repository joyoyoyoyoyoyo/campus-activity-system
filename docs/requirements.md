# 需求说明（Requirements）

## 1. 文档说明

本文件用于明确"校园活动报名与签到管理系统"的功能性、非功能性需求与边界，作为 WBS 拆分及后续开发的依据。

| 版本 | 日期       | 修订人 | 修订说明 |
| ---- | ---------- | ------ | -------- |
| V0.1 | 2026-04-29 | TBD    | 初稿     |

## 2. 项目范围

### 2.1 包含

- 活动信息的发布、修改、下架
- 学生在线浏览与报名
- 报名名单的审核与查看
- 现场签到与签到状态维护
- 签到统计结果导出

### 2.2 不包含

- 复杂支付功能
- 与统一身份认证系统深度对接
- 高并发优化与正式部署
- 移动端原生 App

## 3. 角色与权限

| 角色         | 主要权限                                                       |
| ------------ | -------------------------------------------------------------- |
| 活动管理员   | 创建/修改/下架活动；查看报名与签到统计；导出统计               |
| 学生         | 浏览活动；提交/取消报名；查询个人报名结果                      |
| 现场签到人员 | 通过二维码或学号核验学生身份；登记签到状态                     |
| 系统维护人员 | 维护字典、参数、用户基础信息                                   |

## 4. 功能性需求

### FR-01 活动信息管理（活动管理员）

- FR-01-01 创建活动：标题、简介、时间、地点、报名截止时间、人数上限。
- FR-01-02 修改活动：在报名未开始或未截止前可修改关键字段。
- FR-01-03 下架/取消活动。
- FR-01-04 活动列表按时间倒序展示。

### FR-02 学生报名（学生）

- FR-02-01 浏览活动列表与活动详情。
- FR-02-02 在报名截止时间前提交报名。
- FR-02-03 在审核通过前可取消报名。
- FR-02-04 查询个人历史报名记录。

### FR-03 报名审核（活动管理员）

- FR-03-01 查看某活动报名名单。
- FR-03-02 审核：通过 / 拒绝（拒绝需填写原因）。
- FR-03-03 当报名人数达到上限时停止接收新报名。

### FR-04 签到管理（现场签到人员）

- FR-04-01 通过学号或二维码识别学生身份。
- FR-04-02 登记签到状态：已签到 / 迟到 / 缺席。
- FR-04-03 同一活动同一学生不允许重复签到。

### FR-05 统计与导出（活动管理员）

- FR-05-01 按活动统计：报名人数、实际签到人数、签到率。
- FR-05-02 导出报名名单及签到结果（CSV / Excel）。

### FR-06 系统维护（系统维护人员）

- FR-06-01 用户与角色基础信息维护。
- FR-06-02 系统参数（如默认报名截止提前量）维护。

## 5. 非功能性需求

| 编号  | 类别     | 要求                                                       |
| ----- | -------- | ---------------------------------------------------------- |
| NFR-01 | 易用性   | 主要操作 ≤ 3 步可完成                                      |
| NFR-02 | 可靠性   | 报名与签到不丢数据，关键操作有日志                         |
| NFR-03 | 可维护性 | 代码与文档同仓库，结构清晰，变更可追溯                     |
| NFR-04 | 安全性   | 角色权限隔离，避免越权访问                                 |
| NFR-05 | 性能     | 校内场景下，常规列表查询响应时间 ≤ 2s                       |

## 6. 约束与假设

- 仅在校园网或测试环境运行，不涉及真实身份认证对接。
- 数据规模为小型（百级活动 / 千级用户）。
- 实验项目，不需要部署上线。
